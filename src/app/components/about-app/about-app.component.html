About this test:

At first I found the description was quite abstract but I managed to interpret an idea to have two data tables showing and one table to check if their values match each other.

First thing I built was a component to actually convert the csv to json as I thought this would be required, but after an email to clarify that wasn't needed I just converted the csv to json and started work. The page is split into Sound Inputs Table and Sound Recordings Database Table, details below:

Part 1: Matching candidates

I considered making the app the search and show each item in the inputs json but I thought that would be tedius for the user to check each item one by one, so I simply created a table that shows matches with coloured cells. I match the values to the main database table, if there is match I save that index to an array, then I see if that index matches current index, if yes, it adds class style to change that color. Those styles have a toggle tick box, just to be more interactive.

How would I improve the matching?

  • Add an edit feature to edit the input and database records, I saw some typos like "Mose" and "Fredie Freeloader"
  • If we had access to the sound audio file, play the sound audio
  • Add some unit tests for the startMatchChecker() function, as this is kind of the engine of the app
Part 2: Search the whole database

I am using ngrx to save the json files to the state, I had to create a createSelector() function to filter the state based on the type event in the input.

I thought it is best to have a drop down to select what field to search by.

How would I improve the search?

  • Add validation to the Duration field to only accept numbers
  • Consider using a buffer so there is a pause before the search function runs, this helps if we are doing an api call in each typed input
Part 3: Create a new sound recording in the database

In the Inputs table I just created a button to Add to the DB, this then fires a function that adds that records. With the help of ngrx, I am able to subscibe to changes to the database and immediately update the Inputs Table match color styling.

How would I improve this?

  • Add a feature to create a fresh Sound record from scratch, like a button with "Add new record", which opens a dialog with a small form.

Things to impress:

  • I'm using NGRX, which is tricky to understand at first, simple things can become hard when we deal with immutable state and pure functions!
  • The adding and removing on of the Database Table auto updates the Inputs Table, try removing that ginger talent Ed Sheeran and watch the green cell disapear :D
  • The undo feature on Add to DB, you will see a snackbar component open up and you can undo that action.
