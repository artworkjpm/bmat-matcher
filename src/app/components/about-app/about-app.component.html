About this app:

  • First problem I had was this error message about CORS: No 'Access-Control-Allow-Origin' header is present on the requested resource. > I managed to get around that by using a proxy.conf.json file, NICE!
  • Run the app locally using `npm start`
  • Unit tests:
    My main unit tests are for the ApiService - check GET request for url, favItems array ADD and REMOVE, ItemsComponent - should paginate in sections of 5 , FavIconComponent - should return false if dummyItem is not in dummyFavItems (to control the styling of the heart icon)

Things to impress:

  • I added "lowest" and "highest" price filter
  • The app is mobile responsive, show/hiding a dropdown component for the search component on mobile
  • The app doesn't use a complicated state management, like ngrx or redux, although they are very good. I simply save the state in the ApiService
